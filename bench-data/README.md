# Field microbench artifacts

Generated by `scripts/field_microbench_collect.py` from Criterion baselines under `target/criterion/`.

`ns_per_op_or_lane` / `ns/lane` are taken from Criterion `estimates.json` medians (already in nanoseconds for `WallTime`).

### CSV columns

| Column | Meaning |
|--------|---------|
| `workload` | `latency_chain` (dependent chain) or `throughput_stream` (parallel independent streams) |
| `vectorization` | `scalar` or `packed` (SIMD lanes; `width` is lane count for packed rows) |
| `simd` | Target ISA profile (`neon`, `avx2`, `avx512`) |

Each `(library, field, op, workload, vectorization, arch, simd)` tuple is one row; duplicate `mul` rows differ by `workload`, not by accident.

## Captured baselines

| Baseline | Arch | SIMD | Status |
|----------|------|------|--------|
| `neon` | aarch64 | NEON | captured locally (`RUSTFLAGS="-Ctarget-cpu=native"`) |
| `avx2` | x86_64 | AVX2 | run on x86 server: `RUSTFLAGS="-Ctarget-cpu=x86-64-v3"` |
| `avx512` | x86_64 | AVX-512 | run on x86 server: `RUSTFLAGS="-Ctarget-cpu=native"` (host with AVX-512) |

`cargo bench -p akita-pcs --bench field_arith --no-run` has been verified under both x86 `RUSTFLAGS` configurations.

## Commands

```bash
# NEON (aarch64 workstation)
RUSTFLAGS="-Ctarget-cpu=native" \
  cargo bench -p akita-pcs --bench field_arith -- --save-baseline neon \
  'field_arith/(base|ext4|ext5)/'

# x86_64 AVX2
RUSTFLAGS="-Ctarget-cpu=x86-64-v3" \
  cargo bench -p akita-pcs --bench field_arith -- --save-baseline avx2 \
  'field_arith/(base|ext4|ext5)/'

# x86_64 AVX-512
RUSTFLAGS="-Ctarget-cpu=native" \
  cargo bench -p akita-pcs --bench field_arith -- --save-baseline avx512 \
  'field_arith/(base|ext4|ext5)/'

python3 scripts/field_microbench_collect.py \
  --baseline neon:aarch64:neon \
  --baseline avx2:x86_64:avx2 \
  --baseline avx512:x86_64:avx512
```
